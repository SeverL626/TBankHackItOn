# Meventur

Telegram-бот и Mini App для организации мероприятий: создание событий, запись участников, приватные групповые события, рассылки, оплата и напоминания.

## Что умеет

- Создание мероприятий через кнопки в чате и через Mini App.
- Просмотр ленты, запись и выход из мероприятия через кнопки в чате и Mini App.
- Управление событием в чате: редактирование базовых полей, участники, уведомления, снятие с публикации.
- Админка организатора в Mini App: создание, редактирование, удаление, статистика и уведомления участникам.
- Добавление бота в групповой чат и создание мероприятия группы через личку с привязкой к группе.
- Автоматическая запись упомянутых участников группы, если они уже писали боту `/start`.
- Публичные и приватные мероприятия.
- Уведомления о входе/выходе участников: личные события пишут организатору, групповые события пишут в группу.
- Кастомные уведомления от организатора: в личных событиях участникам, в групповых событиях в группу.
- Кастомные напоминания перед событием: хоть за час, хоть за 5 секунд.
- Технические уведомления о новых деплоях по подписке.
- Сценарий оплаты заранее через СБП с подтверждением организатором.
- Фоновые напоминания за 24 часа, за 2 часа и за 15 минут до начала.
- Автоочистка служебных сообщений в группах, чтобы чат не превращался в простыню.

## Основной принцип UX

Функционал дублируется двумя входами:

- **Telegram-команды и кнопки**: полный сценарий прямо в чате с ботом.
- **Mini App**: визуальный интерфейс для просмотра, участия, создания, редактирования и администрирования.

Бот всегда может предложить Mini App как более удобный вариант, но не должен требовать его: создание, регистрация, уведомления и управление доступны через чатовые кнопки.

Mini App открывается только по HTTPS. `WEBAPP_URL` обязательно должен начинаться с `https://`, иначе Telegram не покажет кнопку Web App.

## Команды бота

| Команда | Что делает |
|---|---|
| `/start` | регистрирует пользователя и показывает меню |
| `/help` | справка |
| `/events` | компактная лента публичных мероприятий |
| `/my` | мои мероприятия и управление событиями организатора |
| `/new` | создание мероприятия через кнопки: публичное/приватное, свободная запись/по приглашению |
| `/broadcast` | кастомное уведомление участникам своего мероприятия |
| `/stats` | статистика и кнопка открытия Mini App |
| `/updates_on` | включить технические уведомления о новых деплоях |
| `/updates_off` | выключить технические уведомления |
| `/cancel` | отмена текущего сценария |

В личке после `/start` появляется постоянная клавиатура: найти, создать, мои, Mini App, уведомление, помощь.

В группе личные команды не выполняют личные сценарии. `/events` открывает события текущей группы, `/new` открывает создание события этой группы, а `/help`, `/stats`, `/my`, `/broadcast` в группе не засоряют чат.

## Mini App

Mini App живет в `src/main/resources/webapp/index.html`, API - в `src/main/kotlin/com/meventus/webapp/WebAppServer.kt`.

Вкладки:

- **EVENTS**: список доступных мероприятий, поиск, фильтр по тегам, вход/выход.
- **MINE**: события, где пользователь участник или организатор.
- **ADMIN**: мероприятия организатора, суммарная статистика, создание, редактирование и уведомления.

Mini App использует Telegram `initData` и проверяет подпись на сервере. Поэтому действия, где нужен пользователь, работают только при открытии из Telegram.

## Публичные и приватные события

### Публичные

- Видны в общем списке `/events` и во вкладке `EVENTS`.
- Любой пользователь может записаться.
- Можно фильтровать по тегам.

### Приватные

- Не попадают в общий публичный список.
- Видны организатору и уже записанным участникам.
- Создаются через `/new` с кнопкой `Приватное`, Mini App или из группы.
- Случайный пользователь не может записаться в приватное событие через Mini App.

## Создание из группы

Бота можно добавить в Telegram-группу и настроить под конкретную команду. В группе он работает не как общая афиша, а как календарь этого чата.

В группе бот работает только как календарь этой группы. Он не показывает Mini App, общую помощь и личную админку, чтобы не засорять чат.

Когда бот добавляется в группу, он отправляет короткое приветствие и предупреждает: всем участникам нужно один раз написать боту `/start` в личку. Это нужно, чтобы бот мог приглашать людей по `@username`, записывать их и отправлять личные уведомления.

Команды группы:

| Команда | Что делает |
|---|---|
| `/gevents` | мероприятия только текущей группы |
| `/gnew` или `/new` | открыть создание в личке, с привязкой к этой группе |
| `/ginvite` | пригласить людей в уже созданное событие |
| `/updates_on` | включить тех-уведомления в группе |
| `/updates_off` | выключить тех-уведомления в группе |

Также можно тегнуть бота: `@meventur_bot мероприятия`, `@meventur_bot создать мероприятие`.

### Простое создание

В группе не нужно вводить все поля одной строкой. Команда `/gnew` отправляет кнопку “Создать в личке”, дальше бот задаёт вопросы в личном чате:

1. видимость: публичное или приватное для группы;
2. режим записи: свободная или по приглашению;
3. название, описание, место, дата, цена, оплата, фото, теги.

После создания бот присылает в группу короткое уведомление. Запись и просмотр идут через `/gevents`.

### Приглашения

Сначала создайте событие, затем пригласите людей по ID события:

```text
/ginvite 12 @user1 @user2
```

ID события показывается в списке `/gevents`.

Правила:

- Дата: `ДД.ММ.ГГГГ ЧЧ:ММ`.
- Участники приглашаются через `/ginvite ID @username`.
- `/gevents` показывает только события текущего группового чата, одним сообщением со страницами.
- Пользователь будет записан автоматически только если он уже написал боту `/start`. Telegram не отдает ID пользователя по обычному текстовому `@username`, поэтому бот ищет его в своей базе.

### Автоочистка в группах

Чтобы не спамить в рабочем чате:

- команды пользователя вроде `/gevents`, `/gnew`, `/ginvite` удаляются через короткое время;
- ошибки и подсказки удаляются автоматически;
- карточки мероприятий живут дольше, чтобы участники успели нажать кнопки.

Если бот не админ группы и не имеет права удалять сообщения, очистка чужих сообщений может не сработать. Свои сообщения бот всё равно старается удалить.

## Уведомления

Автоматически отправляются:

- организатору, когда участник записался в личное событие;
- организатору, когда участник вышел из личного события;
- в группу, когда участник записался или вышел из группового события;
- участнику, если его добавили из группового чата;
- организатору или в группу, когда участник отправил подтверждение оплаты;
- участнику, когда оплата подтверждена или отклонена;
- всем участникам и организатору за 24 часа, за 2 часа и за 15 минут до начала личного события;
- в группу за 24 часа, за 2 часа и за 15 минут до начала группового события;
- всем участникам или в группу при ручном уведомлении от организатора.
- по кастомным правилам организатора: например `1ч`, `30м`, `5с` до начала.

Кастомное напоминание добавляется из управления событием: `/my` -> `Управлять` -> `⏰ Напоминание`.

Напоминания хранят отметки отправки в памяти процесса. После рестарта приложения уже отправленные окна могут быть отправлены повторно, если событие всё ещё попадает в окно напоминания.

## Оплата

При создании события можно выбрать:

- **На месте** - участник сразу записывается.
- **Заранее через СБП** - бот отправляет участнику реквизиты и кнопку `Я перевёл деньги`; организатор получает запрос и подтверждает или отклоняет оплату.

В Mini App создание и редактирование тоже поддерживают тип оплаты, телефон СБП и имя получателя.

## API Mini App

| Метод | Путь | Описание |
|---|---|---|
| `GET` | `/` | HTML Mini App |
| `GET` | `/api/events` | список доступных событий |
| `GET` | `/api/event?id=...` | карточка события |
| `POST` | `/api/join` | записаться |
| `POST` | `/api/leave` | выйти |
| `GET` | `/api/admin` | события организатора |
| `POST` | `/api/event/create` | создать событие |
| `POST` | `/api/event/update` | изменить событие |
| `POST` | `/api/event/delete` | удалить событие организатора |
| `POST` | `/api/broadcast` | рассылка участникам |
| `GET` | `/api/stats` | статистика сообщений |
| `GET` | `/api/photo` | прокси Telegram-фото |

## Архитектура

```text
Telegram Bot API
   |
   v
Meventur Bot
   |-- commands/
   |-- handlers/
   |-- callbacks/
   |-- notifications/
   |
   v
domain services
   |-- EventService
   |-- ParticipantService
   |-- UserService
   |
   v
repositories
   |
   v
PostgreSQL

Mini App WebView
   |
   v
WebAppServer
   |
   v
same domain services
```

Главные файлы:

- `Application.kt` - старт БД, Mini App сервера и Telegram-бота.
- `MeventusBot.kt` - регистрация команд, callback-ов, FSM и напоминаний.
- `WebAppServer.kt` - HTTP API для Mini App.
- `EventCreateHandler.kt` - пошаговое создание события в боте.
- `GroupEventHandler.kt` - создание события из группового чата.
- `EventReminderService.kt` - фоновые напоминания.
- `EventsTable.kt` - схема событий, включая `visibility`, `registration_mode` и `group_chat_id`.

## База данных

Основные таблицы:

- `users` - пользователи Telegram.
- `events` - мероприятия.
- `event_tags` - теги мероприятий.
- `participants` - участники и взносы.

Важные поля `events`:

- `visibility`: `PUBLIC` или `PRIVATE`.
- `registration_mode`: `FREE` или `INVITE_ONLY`.
- `group_chat_id`: ID группы, если событие создано из группового чата.
- `payment_type`: `ON_SITE` или `ADVANCE`.
- `sbp_phone`, `sbp_name`: реквизиты для оплаты заранее.

Дополнительные таблицы:

- `custom_reminders` - кастомные напоминания организатора.
- `tech_update_subscriptions` - чаты, подписанные на тех-уведомления.
- `app_state` - служебное состояние приложения, например последний отправленный релиз.

Таблицы создаются и расширяются автоматически при старте через Exposed `SchemaUtils.createMissingTablesAndColumns`.

## Конфигурация

Переменные окружения:

```env
BOT_TOKEN=...
BOT_USERNAME=meventur_bot

POSTGRES_DB=meventus
POSTGRES_USER=meventus
POSTGRES_PASSWORD=...

DATABASE_URL=jdbc:postgresql://db:5432/meventus
DATABASE_USER=meventus
DATABASE_PASSWORD=...

WEBAPP_HOST=185.4.72.98.sslip.io
WEBAPP_PORT=8080
WEBAPP_URL=https://185.4.72.98.sslip.io
RELEASE_ID=<git sha или версия деплоя>
```

Для Telegram Mini App критично:

```env
WEBAPP_URL=https://...
```

Если там `http://localhost:8080`, пусто или просто IP без HTTPS, кнопка Mini App в Telegram не появится.

Для тех-уведомлений при деплое нужен `RELEASE_ID`. В GitHub Actions он прокидывается как `${{ github.sha }}`. Если `RELEASE_ID`/`GITHUB_SHA`/`APP_VERSION` не задан, бот не рассылает сообщение “Вышло обновление!” при старте, чтобы не спамить на обычных рестартах.

## Локальный запуск

```bash
docker compose up -d db
./gradlew run
```

Для теста Mini App в Telegram нужен HTTPS-туннель:

```bash
cloudflared tunnel --url http://localhost:8080
```

Затем указать выданный URL в `WEBAPP_URL` и перезапустить приложение.

## Деплой

Сервис запускается через Docker Compose:

- `app` - Kotlin-приложение, порт `127.0.0.1:8080`.
- `db` - PostgreSQL.
- `caddy` - HTTPS reverse proxy на `WEBAPP_HOST`.

Проверки на VPS:

```bash
cd /opt/myapp
docker compose ps
docker compose logs app --tail 100
docker exec tbank-hackiton-app printenv WEBAPP_URL
curl -I http://localhost:8080
curl -I https://185.4.72.98.sslip.io
```

Если Caddy не стартует с ошибкой `address already in use`, значит порт `80` или `443` занят другим процессом:

```bash
sudo ss -ltnp | grep -E ':80|:443'
```

Обычно виноват `nginx`, `apache2` или другой `caddy`.

## GitHub Actions

Автодеплой ожидает SSH-доступ к серверу. В Secrets репозитория нужны:

- `VPS_HOST`
- `VPS_USER`
- `VPS_SSH_KEY`
- `VPS_SSH_PORT`

Deploy key на GitHub нужен серверу, чтобы делать `git pull` из репозитория. Public key добавляется в GitHub: **Settings -> Deploy keys -> Add deploy key**. Private key остается на сервере или в GitHub Actions secret, в зависимости от выбранной схемы деплоя.

## Проверка перед релизом

```bash
./gradlew test
docker compose up -d --build
docker compose logs app --tail 100
```

Минимальная ручная проверка:

1. `/start` у нового пользователя.
2. `/new` и создание публичного/приватного события через кнопки.
3. Открыть Mini App через `/stats`.
4. Создать событие в `ADMIN`.
5. Записаться на публичное событие через Mini App.
6. Добавить бота в группу и создать событие через `/gnew`.
7. Проверить `/gevents`, запись участника, `/ginvite`.
8. Проверить уведомление `/broadcast` и из `ADMIN`.
